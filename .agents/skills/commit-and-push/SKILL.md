# commit-and-push

## When to use
This skill can be invoked using its original name but also the keyword "camp"

## Instructions
1. Check for any unstaged changes, stage them and commit the code with a descriptive message, the message should encapsulate the changes but do not include to much technica jargon. After committing, push the changes on the currently active branch.
